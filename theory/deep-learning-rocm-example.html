ROCm で見るシリーズ

小さなニューラルネットを PyTorch ROCm で 1 回動かし、Linear 層・ReLU・出力がどう現れるか見ます。

Run a tiny neural network once on PyTorch ROCm and see how Linear layers, ReLU, and outputs appear.

📘 対応する理論ページ: イメージでわかる深層学習 — 畳み込み・フィルタ・solver を比喩で解説

📘 Theory page: Visual Deep Learning — Convolution, filters, and solvers through analogy

1. なにをするか

1. What We'll Do

2層の全結合ネットワーク（Linear → ReLU → Linear）を作り、ランダムな入力を1回通します。「ニューラルネットが答えを出す」最小の例です。

Build a 2-layer fully-connected network (Linear → ReLU → Linear) and pass random input through it once. This is the smallest example of "a neural network producing output."

2. 最小コード

2. Minimal Code

import torch import torch.nn as nn # 小さなニューラルネット: 入力4 → 隠れ8 → 出力2 model = nn.Sequential( nn.Linear(4, 8), nn.ReLU(), nn.Linear(8, 2), ).to("cuda") # ランダムな入力 (バッチ1, 特徴4) x = torch.randn(1, 4).to("cuda") # forward: ネットに通して答えを得る y = model(x) print("入力 x:", x.cpu()) print("出力 y:", y.cpu().detach()) print("モデル構造:") print(model)

3. 実行するとどうなるか

3. What Happens When You Run It

入力 x: tensor([[ 0.3171, -1.0434, 0.5629, -0.2218]]) 出力 y: tensor([[ 0.1283, -0.0652]]) モデル構造: Sequential( (0): Linear(in_features=4, out_features=8, bias=True) (1): ReLU() (2): Linear(in_features=8, out_features=2, bias=True) )

4 つの数字が入って、2 つの数字が出てきました。間で起きていることは:

4 numbers went in, 2 numbers came out. What happened in between:

① Linear(4→8): 入力 4 個を行列積で 8 個に変換（rocBLAS GEMM）
② ReLU: マイナスの値を 0 にする（GPU kernel）
③ Linear(8→2): 8 個を行列積で 2 個に変換（rocBLAS GEMM）

① Linear(4→8): Transform 4 inputs to 8 via matrix multiply (rocBLAS GEMM)
② ReLU: Zero out negative values (GPU kernel)
③ Linear(8→2): Transform 8 to 2 via matrix multiply (rocBLAS GEMM)

🏭 理論ページとの対応: 理論ページの「フィルタで大事な特徴を見つける → 通さない値はそのまま無視 → 最終判定」が、ここでは Linear → ReLU → Linear として現れています。

🏭 Theory page connection: The theory page's "find features with filters → ignore irrelevant values → final judgment" appears here as Linear → ReLU → Linear.

4. 裏では何が起きているか

4. Under the Hood

各 Linear 層は内部で y = Wx + b（行列積 + バイアス加算）を実行しています。PyTorch ROCm はこれを rocBLAS に委ねます。ReLU は単純な要素ごとの操作なので、独自の GPU kernel で処理されます。

Each Linear layer internally computes y = Wx + b (matrix multiply + bias). PyTorch ROCm delegates this to rocBLAS. ReLU is a simple element-wise operation handled by a dedicated GPU kernel.

畳み込み（Conv2d）を使う場合は、rocBLAS ではなく MIOpen が担当します。MIOpen は最適な solver を自動選択して畳み込みを実行します。

If you use convolution (Conv2d), MIOpen takes over instead of rocBLAS. MIOpen automatically selects the best solver to run the convolution.
