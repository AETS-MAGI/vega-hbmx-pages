Run one epoch of forward → loss → backward → optimizer.step() on PyTorch ROCm and confirm that weights change.

Note: GPU selection appears as "cuda" in this page, but that is the compatibility API name used by PyTorch ROCm. Execution path is HIP/ROCm.

1. なにをするか

1. What We'll Do

小さなネットで「正解に近づける」を 1 ステップだけ実行します。重みの before / after を見て、学習が起きたことを確認します。

Run one "get closer to the correct answer" step with a small network. By comparing weights before and after, we confirm that learning happened.

2. 最小コード

2. Minimal Code

import torch import torch.nn as nn assert torch.version.hip is not None, "This page expects PyTorch ROCm build" DEVICE = torch.device("cuda") # 1. モデル: 入力3 → 出力1 model = nn.Linear(3, 1).to(DEVICE) # 2. 正解データ（ダミー） x = torch.tensor([[1.0, 2.0, 3.0]]).to(DEVICE) target = torch.tensor([[7.0]]).to(DEVICE) # 3. 損失関数とオプティマイザ loss_fn = nn.MSELoss() optimizer = torch.optim.SGD(model.parameters(), lr=0.01) # 学習前の重みを記録 w_before = model.weight.data.clone().cpu() # 4. 学習 1 ステップ pred = model(x) # forward loss = loss_fn(pred, target)# loss 計算 loss.backward() # backward (勾配計算) optimizer.step() # 重み更新 optimizer.zero_grad() # 勾配リセット # 学習後の重み w_after = model.weight.data.clone().cpu() print("loss:", loss.item()) print("重み (前):", w_before) print("重み (後):", w_after) print("差分:", w_after - w_before)

3. 実行するとどうなるか

3. What Happens When You Run It

loss: 32.4198 重み (前): tensor([[-0.0043, 0.2987, -0.1835]]) 重み (後): tensor([[ 0.1034, 0.5141, -0.0298]]) 差分: tensor([[0.1077, 0.2154, 0.1537]])

重みが変わりました。これが「1 回学習した」ということです。loss が大きいほど重みの変化も大きくなります。

Weights changed — that's "one step of learning." The larger the loss, the larger the weight change.

理論ページとの対応:
forward = model(x) — 入力をネットに通す
loss = 予測と正解のズレを数値化
backward = loss.backward() — 勾配を計算
optimizer.step() = 重みを少し修正

Theory page mapping:
forward = model(x) — pass input through the network
loss = quantify the gap between prediction and target
backward = loss.backward() — compute gradients
optimizer.step() = adjust weights a little

4. 裏では何が起きているか

4. Under the Hood

forward: rocBLAS GEMM で行列積を実行。
backward: 自動微分グラフを逆順にたどり、各パラメータの勾配を計算。計算の大部分はやはり rocBLAS。
optimizer.step(): SGD の場合は w = w - lr × grad の単純な kernel。

forward: rocBLAS GEMM executes the matrix multiply.
backward: Traverses the autograd graph in reverse, computing gradients for each parameter — again mostly rocBLAS.
optimizer.step(): For SGD, a simple kernel: w = w - lr × grad.

学習は推論より重く、forward だけでなく backward と update が追加されます。ROCm 観点では、行列演算回数と中間テンソル保持が増えるため、メモリ圧と計算時間が上がります。

Training is heavier than inference because it adds backward and update on top of forward. From a ROCm perspective, this increases matrix operations and intermediate tensor retention, raising memory pressure and compute time.

5. ROCmで観測するポイント

5. What to Observe on ROCm

# ROCm ビルド確認 python -c "import torch; print('torch.version.hip =', torch.version.hip)" # 学習ステップの HIP 呼び出し観測 rocprof --hip-trace python training_step_example.py

同じモデルで 推論ページ の実行時間と比較すると、学習側が重いことを観測できます。理論ページの「学習は重い」を実測でつなぐ目的です。

Compare runtime with the same model on the inference page to observe why training is heavier. This connects the theory claim to actual measurement.

🎯 ポイント: 学習は forward + backward + update の繰り返しです。ここでは 1 回だけ実行しましたが、実際には数万〜数十万回繰り返して loss を下げ続けます。

🎯 Key point: Training is the repetition of forward + backward + update. We ran it once, but in practice this repeats thousands to millions of times to keep reducing loss.
