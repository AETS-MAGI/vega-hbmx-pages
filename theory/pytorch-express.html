What is PyTorch? in one page

このページは「PyTorch が何者か」を最短でつかむための導入です。深層学習・学習フロー・推論フロー・ROCm 実例ページに入る前の橋渡しとして使えます。

This page is a fast introduction to what PyTorch is. Use it as a bridge before deep learning, training flow, inference flow, and ROCm example pages.

1. PyTorch は何か

1. What PyTorch Is

ひとことで言うと、テンソル計算 + 自動微分（勾配を自動で出す） + 深層学習部品のセットです。

In one sentence: tensor math + autograd + deep-learning building blocks.

PyTorch は、数値計算ライブラリであり、同時に深層学習フレームワークです。NumPy 風のテンソル操作でモデルを書き、必要な勾配（どちらへ直せばよいかの情報）を自動計算して学習まで進められます。

PyTorch is both a numerical computing library and a deep-learning framework. You write models with NumPy-like tensor operations, then autograd computes gradients for training.

覚える最小セット: Tensor（データの箱） / Module（部品） / autograd（自動で勾配計算） / optimizer（直し方を決める係）

Minimal set to remember: Tensor / Module / autograd / optimizer

3. 最短サンプル（10行）

3. Shortest Sample (10 lines)

import torch import torch.nn as nn x = torch.randn(1, 4) model = nn.Sequential( nn.Linear(4, 8), nn.ReLU(), nn.Linear(8, 2), ) y = model(x) print(x.shape, y.shape)

この10行に「テンソルを作る」「層を並べる」「forwardする」が入っています。学習を足す場合は loss / backward / optimizer.step を追加します。

These 10 lines already contain: create tensors, stack layers, run forward. To train, add loss / backward / optimizer.step.

4. ROCm で使うときの最初の注意

4. First ROCm Notes

注記: PyTorch ROCm ではデバイス指定に "cuda" 文字列を使う場合があります。これは API 互換上の名前で、実行経路は HIP/ROCm 側です。

Note: In PyTorch ROCm, device selection may still use the string "cuda". This is an API-compatibility name; execution paths are HIP/ROCm.

import torch print("torch.version.hip:", torch.version.hip) print("cuda available:", torch.cuda.is_available()) if torch.cuda.is_available(): print("device:", torch.cuda.get_device_name(0))

5. ここで詰まりやすいポイント

5. Common First Stumbles

1) Python バージョンと wheel の対応がずれる
2) GPU は見えていても PyTorch が未導入
3) 学習と推論の違い（重みが変わるか）を混同する

1) Python version and wheel compatibility mismatch
2) GPU visible but PyTorch not installed
3) Mixing up training vs inference (whether weights change)
