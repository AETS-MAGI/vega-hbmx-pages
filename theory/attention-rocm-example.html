Scaled Dot-Product Attention を手計算し、nn.MultiheadAttention と比較してしくみを確認します。

Compute Scaled Dot-Product Attention by hand, then compare with nn.MultiheadAttention.

📘 対応する理論ページ: イメージでわかる Attention — Q / K / V の役割とスケーリングのしくみ

📘 Theory page: Visual Attention — Roles of Q / K / V and how scaling works

1. なにをするか

1. What We'll Do

3 単語 × 4 次元のダミー入力で、Scaled Dot-Product Attention を forward します。注目度（attention weights）がどのように分布するかを見ます。

Run Scaled Dot-Product Attention on a dummy input of 3 tokens × 4 dimensions. Observe how attention weights distribute across tokens.

2. 手計算で見る Attention

2. Attention by Hand

import torch import torch.nn.functional as F import math assert torch.version.hip is not None, "This page expects PyTorch ROCm build" DEVICE = torch.device("cuda") # 3トークン, 次元4 のダミー入力 x = torch.tensor([ [1.0, 0.0, 1.0, 0.0], # トークン A [0.0, 1.0, 0.0, 1.0], # トークン B [1.0, 1.0, 0.0, 0.0], # トークン C ]).to(DEVICE) # Self-attention: Q = K = V = x (簡略版) Q, K, V = x, x, x d_k = Q.size(-1) # = 4 # Scaled Dot-Product Attention scores = torch.matmul(Q, K.T) / math.sqrt(d_k) weights = F.softmax(scores, dim=-1) output = torch.matmul(weights, V) print("スコア (QK^T / √d):") print(scores.cpu()) print("注目度 (softmax):") print(weights.cpu()) print("出力:") print(output.cpu())

3. 実行するとどうなるか

3. What Happens When You Run It

スコア (QK^T / √d): tensor([[1.0000, 0.0000, 0.5000], [0.0000, 1.0000, 0.5000], [0.5000, 0.5000, 1.0000]]) 注目度 (softmax): tensor([[0.4219, 0.1553, 0.2561], [0.1553, 0.4219, 0.2561], [0.2312, 0.2312, 0.4810]]) 出力: tensor([[0.6780, 0.4114, 0.4219, 0.1553], [0.4114, 0.6780, 0.1553, 0.4219], [0.4624, 0.4624, 0.2312, 0.2312]])

注目度の行列は各トークンが他のトークンに「どれくらい注目するか」を示しています。トークン A は自分自身 (0.42) に一番注目し、トークン B (0.16) より トークン C (0.26) に多く注目しています。

The attention weight matrix shows how much each token "pays attention" to others. Token A attends most to itself (0.42), more to Token C (0.26) than Token B (0.16).

理論ページとの対応:
Q (Query): 「何を探したいか」 — 各トークンの問い合わせベクトル
K (Key): 「何を持っているか」 — 各トークンの識別ベクトル
V (Value): 「実際に渡す情報」 — 注目度で重み付けされる
√d_k で割る: スコアが大きくなりすぎて softmax がほぼ 0/1 に偏るのを防ぐ

Theory page mapping:
Q (Query): "What am I looking for?" — each token's query vector
K (Key): "What do I have?" — each token's identifier vector
V (Value): "What to actually pass" — weighted by attention scores
Divide by √d_k: Prevents scores from getting so large that softmax collapses to near-0/1

4. nn.MultiheadAttention で同じことをする

4. Same Thing with nn.MultiheadAttention

import torch.nn as nn # MultiheadAttention: 1ヘッド, 次元4 mha = nn.MultiheadAttention( embed_dim=4, num_heads=1, batch_first=True ).to(DEVICE) # 入力を (batch=1, seq=3, dim=4) に x_batch = x.unsqueeze(0) with torch.no_grad(): out, attn_weights = mha(x_batch, x_batch, x_batch) print("MHA 注目度:") print(attn_weights.squeeze().cpu()) print("MHA 出力 shape:", out.shape)

nn.MultiheadAttention は内部で Q, K, V へ線形変換してから Scaled Dot-Product を実行します。手計算版と数値は違いますが、構造は同じです。

nn.MultiheadAttention applies linear projections to create Q, K, V before the Scaled Dot-Product. Numbers differ from our hand-computed version, but the structure is the same.

5. 裏では何が起きているか

5. Under the Hood

QKT の計算は行列積なので rocBLAS GEMM が使われます。softmax は専用 kernel。weights × V もまた rocBLAS GEMM です。つまり Attention 1 回で rocBLAS GEMM が少なくとも 2 回呼ばれます。

QKT is a matrix multiply → rocBLAS GEMM. Softmax has a dedicated kernel. weights × V is another rocBLAS GEMM. So one Attention call invokes rocBLAS GEMM at least twice.

Transformer の各層にこの Attention が 1 つ以上あり、層数 × ヘッド数の回だけ繰り返されます。これが LLM が大量の行列演算を必要とする理由です。

Each Transformer layer has one or more Attention blocks, repeated across layers × heads. This is why LLMs require massive matrix computation.

6. ROCmで観測するポイント

6. What to Observe on ROCm

# ROCm ビルド確認 python -c "import torch; print('torch.version.hip =', torch.version.hip)" # Attention 実行時の HIP 呼び出し観測 rocprof --hip-trace python attention_example.py

🏭 マルチヘッドの意味: 1 つの Attention は 1 種類の関係しか学べません。MultiheadAttention は複数の「視点」から同時に注目し、結果を結合します（理論ページでは「複数の虫眼鏡」に例えています）。

🏭 Why multiple heads? One Attention head can only learn one kind of relationship. MultiheadAttention attends from multiple "viewpoints" simultaneously and concatenates results (the theory page calls them "multiple magnifying glasses").
