古い AMD GPU は、まだ使えるのか？

Can an older AMD GPU still be useful today?

このページは、専門用語をなるべく減らして、今回の調査で何を調べ、何が分かったのかを短くつかめるようにした入口ページです。

This page is a plain-language entry point. It explains what was tested and what was learned, without requiring deep engineering knowledge.

同じ GPU を使って、ROCm/HIP と Vulkan という2つの動かし方を比べました。
The same GPU was tested with two different software paths: ROCm/HIP and Vulkan.
結果は単純な「動く / 動かない」ではなく、バックエンド依存かつモデル依存でした。
The outcome was not just “works / does not work.” It depended on the backend and also on the model.
つまり「古いGPUだから全部だめ」ではなく、どの経路で使うかが重要でした。
So the answer was not “the GPU is simply too old.” What mattered was which software path was used.

何を調べたの？

What was actually tested?

今回の調査は「古い GPU が全面的にだめか」を決めつけるためではなく、どこで問題が起きるのかを切り分けるために行いました。

The goal was not to label the GPU as simply “bad,” but to isolate where the problem appears.

1

同じ条件で比べた

Matched conditions

同じ GPU、同じモデル、同じプロンプト条件で、バックエンドだけを変えて比べました。

The same GPU, model, and prompt conditions were used, while only the backend was changed.

2

言葉の意味も確認した

We checked the parameter meanings

`num_gpu` が「GPU枚数」ではなく「GPUに移す層数」だと、コードを追って確定しました。

The code trace confirmed that `num_gpu` means “layers moved to GPU,” not “number of GPUs.”

3

結果を何度も確かめた

We repeated the runs

1回だけの偶然で終わらないように、2日間にわたって再現確認を行いました。

The experiments were repeated across two days so the result would not depend on a single lucky or unlucky run.

一番大事な結果

The most important result

同じ Vega / gfx900 を使っても、どのソフトウェア経路を通るかで結果が変わりました。

Even with the same Vega / gfx900 GPU, the outcome changed depending on which software path was used.

ROCm/HIP 側

ROCm/HIP side

今回の tested setup では、確認した条件の範囲で完走できました。

In this tested setup, ROCm/HIP completed the checked conditions successfully.

Vulkan 側

Vulkan side

`num_gpu=0` では安定しましたが、`qwen3.5:2b` では GPU offload が入ると失敗しました。

`num_gpu=0` remained stable, but with `qwen3.5:2b`, Vulkan failed once GPU offload became active.

num_gpu=0
5/5 OK
num_gpu=1
Crash
num_gpu=2/-1
Crash
ここが大事です。 結論は「Vega は全部だめ」ではありません。今回の条件では、ROCm/HIP では通る経路と、Vulkan で特定条件下に失敗する経路が分かれました。さらに、別モデルの `phi4-mini` は Vulkan offload でも通ったので、失敗は GPU 全体に普遍的ではなく、モデル依存でもありました。 This is the key point. The conclusion is not “Vega is bad everywhere.” In these tested conditions, one path worked under ROCm/HIP, while another path failed under Vulkan for specific conditions. And because `phi4-mini` survived Vulkan offload, the failure was not universal to the GPU itself; it was also model-dependent.

なぜこの話が大事？

Why does this matter?

古い GPU を持つ学生や小規模ラボにとって、「公式に新しくない」ことと「もう使えない」ことは同じではありません。

For students and small labs, “not officially current” is not the same as “completely unusable.”

ハードだけでは決まらない

Hardware alone did not decide the outcome

同じ GPU でも、ソフトウェア経路が違うと結果が違いました。

The same GPU behaved differently depending on the software path.

証拠で切り分けられる

Evidence can separate the causes

コード、ログ、再現実験を合わせることで、曖昧な印象ではなく具体的な説明に近づけました。

By combining code, logs, and repeated runs, the explanation became evidence-based instead of impression-based.

再利用の可能性が見える

Reuse still has room

古い GPU でも、条件を選べば役立つ可能性があると示せました。

The study suggests that older GPUs may still be useful when the conditions and software path are chosen carefully.
